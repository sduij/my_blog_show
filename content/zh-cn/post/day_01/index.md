1.BufferedReader的父类是以下哪个？
A
FilterReader
B
InputStreamReader
C
PipedReader
D
Reader

答案：D
解析:
    BufferedReader 的直接父类是 Reader，这是 Java IO 体系中的一个重要继承关系。Reader 是一个抽象类，它是所有字符输入流的基类，定义了读取字符流的基本方法。BufferedReader 的设计目的是为了提供缓冲功能，提高字符输入流的读取效率。它通过继承 Reader 类获得基本的字符读取能力，并在此基础上添加了缓冲区功能，这使得它能够一次读取多个字符，减少实际的 I/O 操作次数。

    InputStreamReader 也是 Reader 的子类，它的主要作用是将字节流转换成字符流，但它与 BufferedReader 是平级关系，都直接继承自 Reader。



2.关于JAVA的垃圾回收机制，下面哪些结论是正确？
A
程序可以任意指定释放内存的时间
B
JAVA程序不能依赖于垃圾回收的时间或者顺序
C
程序可明确地标识某个局部变量的引用不再被使用
D
程序可以显式地立即释放对象占有的内存

答案：B
解析：
    Java的垃圾回收是由JVM自动完成的,程序员不能直接控制垃圾回收的具体行为。B选项正确,因为垃圾回收的时间和顺序是由JVM自动决定的,程序不能依赖于特定的垃圾回收时间或顺序。这是Java内存管理的重要特性。

    分析其他选项:

    A错误:程序不能任意指定内存释放的时间,这完全由JVM的垃圾回收器控制。虽然可以调用System.gc()建议进行垃圾回收,但这只是一个建议,不保证立即执行。

    C错误:虽然可以将局部变量设为null表示不再使用,但这并不是"明确标识",因为垃圾回收器最终还是要通过可达性分析来判断对象是否可回收。

    D错误:在Java中程序不能显式释放内存。这与C/C++这样的语言不同,Java的内存释放完全依赖垃圾回收机制自动完成,程序员无法强制立即释放内存。

    这体现了Java的自动内存管理特性,它既减轻了程序员的负担,也避免了手动内存管理容易出现的问题。



3.已知有下列Test类的说明，在该类的main方法的横线处，则下列哪个语句是正确的？（）
public class Test
{
private float f = 1.0f;
int m = 12;
static int n = 1;
public static void main (String args[])
{
Test t = new Test();
————————
}
}
A
t.f = 1;
B
this.n = 1;
C
Test.m = 1;
D
Test.f = 1;

答案：A
解析：
    这道题目考察了Java中静态成员和实例成员的访问规则。A选项是正确的,因为通过对象实例可以访问非静态成员变量。

    分析每个选项:

    A. t.f = 1; 正确。
    - t是Test类的实例对象
    - f是Test类的实例成员变量
    - 可以通过实例对象t来访问实例成员f

    B. this.n = 1; 错误。
    - this关键字只能在非静态方法中使用
    - main方法是静态方法,不能使用this关键字

    C. Test.m = 1; 错误。
    - m是非静态成员变量
    - 不能通过类名直接访问非静态成员
    - 必须通过实例对象访问

    D. Test.f = 1; 错误。
    - f是private修饰的实例成员变量
    - 不能通过类名直接访问非静态成员
    - private成员只能在类内部访问

    总结:在静态方法中访问成员变量需要遵循以下规则:
    1. 静态成员可以通过类名直接访问
    2. 非静态成员必须通过实例对象访问
    3. 静态方法中不能使用this关键字



4.final、finally、finalize三个关键字的区别是（）
A
final是修饰符（关键字）可以修饰类、方法、变量
B
finally在异常处理的时候使用，提供finally块来执行任何清除操作
C
finalize是方法名，在垃圾收入集器将对象从内存中清除出去之前做必要的清理工作
D
finally和finalize一样都是用于异常处理的方法

答案：ABC
解析：
    final、finally和finalize是Java中三个看似相似但用途完全不同的关键字和方法。ABC选项正确地描述了它们各自的特点和用途。

    选项分析：
    A正确：final是Java中的修饰符，具有以下使用场景：
    - 修饰类：表示该类不能被继承
    - 修饰方法：表示该方法不能被重写
    - 修饰变量：表示该变量是一个常量，赋值后不能修改

    B正确：finally是异常处理机制的一部分，用于定义一个代码块，无论try块中是否发生异常，finally块中的代码都会被执行。通常用于释放资源，如关闭文件、数据库连接等。

    C正确：finalize()是Object类中定义的方法，当垃圾收集器准备回收对象所占用的内存之前，会先调用这个方法。可以在这个方法中执行一些清理操作，但不建议依赖该方法，因为其执行时机是不确定的。

    D错误：此选项混淆了finally和finalize的概念。finalize()是与垃圾回收相关的方法，而不是用于异常处理；finally是异常处理机制的一部分，两者功能和使用场景完全不同。
